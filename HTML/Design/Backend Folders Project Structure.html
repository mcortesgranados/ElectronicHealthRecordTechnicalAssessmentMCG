Backend Folder Architecture Mapping

Folder Applied Architecture(s) Justification
controllers/ Hexagonal Architecture (Adapters) Controllers act as entry points for handling HTTP requests and map them to services, following separation of concerns.
services/ Hexagonal, Clean Architecture (Application Layer) This layer contains business logic, making the system flexible, maintainable, and testable.
repositories/ Hexagonal, Clean Architecture (Persistence Layer) Handles data persistence and communication with EHR systems or databases, decoupling data access from business logic.
dtos/ Clean Architecture (Data Layer) Data Transfer Objects (DTOs) ensure structured and validated data between layers, preventing invalid requests.
middlewares/ Microservices, Security Middleware centralizes authentication, logging, and error handling, improving modularity.
utils/ Clean Architecture (Helper Layer) Contains helper functions (formatting, encryption) to avoid code duplication and enhance reusability.
config/ Microservices, Clean Architecture Stores environment variables and global settings, simplifying configuration management.
infra/db/ Hexagonal, Clean Architecture (Persistence Layer) Manages database connections, schemas, and migrations, ensuring a loosely coupled data layer.
infra/kafka/ Event-Based, Microservices Implements asynchronous communication for event-driven operations between services.
infra/redis/ Microservices, Performance Optimization Implements caching to reduce database load, improving system performance.
infra/security/ Security, Clean Architecture Manages encryption, hashing, and JWT authentication, enforcing data privacy.
tests/unit/ Clean Architecture, Testing Strategy Contains unit tests to validate business logic and ensure high test coverage.
tests/integration/ Hexagonal, Testing Strategy Tests interactions between different modules, ensuring proper integration with databases and external APIs.
index.ts Microservices, Clean Architecture Entry point for the backend application, bootstraps all necessary modules while keeping them decoupled.
